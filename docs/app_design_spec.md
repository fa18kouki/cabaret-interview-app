# キャバクラ面接アプリ デザイン仕様書

## 1. 概要

本ドキュメントは、キャバクラ面接アプリの主要画面のデザイン仕様を定義するものです。ユーザー種別（キャスト、店舗、管理者/スカウト）ごとに、UI/UXの方向性と画面構成を定めます。

## 2. 共通デザイン原則

- **UI/UXの優先**: 全ての画面において、直感的で分かりやすい操作性を最優先します。
- **レスポンシブデザイン**: モバイルファーストを基本とし、PCでの利用も考慮したレスポンシブデザインを採用します。
- **一貫性**: フォント、アイコン、ボタンなどの基本的なコンポーネントは、アプリ全体で一貫性を保ちます。

## 3. ユーザー種別ごとのデザイン方針

### 3.1. キャスト（女の子）向け画面

- **コンセプト**: 「安心と期待」。新しいキャリアへの一歩を、不安なく踏み出せるようなデザイン。
- **トーン＆マナー**: LPと一貫性を保ち、明るく清潔感のあるデザイン。女性ファッション誌のような洗練された雰囲気を踏襲。
- **カラー**: 白を基調に、ピンクやパステルカラーをアクセントとして使用。安心感を与える柔らかな配色。

### 3.2. 店舗（店長）向け画面

- **コンセプト**: 「効率と結果」。多忙な店舗運営の合間でも、効率的に採用活動ができる実用的なデザイン。
- **トーン＆マナー**: プロフェッショナルで、落ち着いたデザイン。夜の業界の雰囲気に合わせつつ、モダンで洗練された印象を与えるダークモードを基調とする。
- **カラー**: ダークグレーや紺色をベースに、ゴールドや白をアクセントに使用。情報の視認性を高めるため、コントラストを意識した配色。

### 3.3. 管理者/スカウト向け画面

- **コンセプト**: 「管理と分析」。複数の店舗やキャストの状況を俯瞰し、データに基づいた判断を支援するデザイン。
- **トーン＆マナー**: データドリブンで機能的なデザイン。情報を整理し、可視化することに重点を置く。
- **カラー**: ニュートラルなグレーを基調とし、グラフやステータス表示には識別しやすい色（青、緑、オレンジなど）を使用。

## 4. 主要画面仕様

### 4.1. キャスト向け画面

| 画面名 | 目的 | 主要コンポーネントとUI/UXのポイント |
| :--- | :--- | :--- |
| **登録/ログイン** | SNSやメールで簡単に利用を開始する | - アイコンを大きく配置し、タップしやすいSNS連携ボタンをメインに。<br>- 「メールアドレスで登録」は目立たないように配置。 |
| **AI診断チャット** | 対話形式で希望条件や適性を入力する | - LINEのような親しみやすいチャットUI。<br>- AIからの質問には選択肢を多用し、自由入力の手間を省く。<br>- 診断の進捗がわかるプログレスバーを表示。 |
| **診断結果** | 自分のタイプやおすすめの店舗傾向を知る | - 「あなたは〇〇タイプ」といった診断結果を大きく表示。<br>- レーダーチャートなどで適性を可視化し、ゲーム感覚で楽しめるようにする。<br>- 「結果を詳しく見る」ボタンで店舗検索画面へ誘導。 |
| **店舗検索/一覧** | 希望条件に合う店舗を探す | - 条件（エリア、時給など）での絞り込み機能を上部に固定。<br>- 各店舗はカード形式で表示（写真、店名、主要な特徴、AIマッチ度）。<br>- 「新着」「人気」などのソート機能。 |
| **店舗詳細** | 店舗の詳しい情報を確認する | - 写真を多めに使い、お店の雰囲気を伝える。<br>- 給与、待遇、求める人物像などの情報をタブで整理して表示。<br>- 「気になる」「チャットで質問する」ボタンを分かりやすく配置。 |
| **チャット** | 店舗と匿名でコミュニケーションをとる | - 既読/未読がわかる標準的なチャットUI。<br>- 面接日程の調整を補助する定型文テンプレート機能。 |
| **マイページ** | 登録情報や活動状況を確認・編集する | - プロフィール編集、応募状況、気になるリストなどを一覧表示。<br>- 設定やログアウトへの導線を確保。 |

### 4.2. 店舗向け画面 (ダークモード)

| 画面名 | 目的 | 主要コンポーネントとUI/UXのポイント |
| :--- | :--- | :--- |
| **ダッシュボード** | 採用活動の全体像を把握する | - 「新規応募」「未読メッセージ」「本日の面接」などの重要指標をカード形式で表示。<br>- 新着応募キャストの一覧を時系列で表示。 |
| **キャスト検索** | 求める条件に合うキャストを探す | - 豊富な検索フィルター（Must/Want要件、経験、年齢など）を用意。<br>- 検索結果はリスト形式で表示（顔写真、名前、年齢、AIランク、主要スキル）。<br>- 詳細プロフィールへの導線を明確にする。 |
| **キャスト詳細** | 応募者の詳細情報を確認し、採否を判断する | - プロフィール情報を左側に、活動履歴やAI分析レポートを右側に配置する2カラムレイアウト。<br>- 「スカウト」「不採用」などのアクションボタンを上部に固定。 |
| **求人管理** | 募集内容を作成・管理する | - 現在募集中の求人一覧を表示。<br>- 「新規求人作成」ボタンから、テンプレートを元に簡単に募集要項を作成できる。 |
| **チャット** | 応募者とコミュニケーションをとる | - 応募者とのやり取りに集中できるシンプルなUI。<br>- 応募者ごとにチャット履歴を管理。 |

### 4.3. 管理者/スカウト向け画面

| 画面名 | 目的 | 主要コンポーネントとUI/UXのポイント |
| :--- | :--- | :--- |
| **メインダッシュボード** | サービス全体のKPIを監視する | - 登録者数、マッチング数、売上などの重要指標をグラフやチャートで可視化。<br>- 期間での絞り込みやデータのエクスポート機能。 |
| **ユーザー管理** | キャスト・店舗の情報を管理する | - 全ユーザーを検索・一覧表示。<br>- ユーザー情報の編集、アカウント停止などの管理操作が可能。 |
| **コンテンツ監視** | 不適切な投稿やプロフィールを監視・対応する | - 通報があったコンテンツや、AIが検知した不適切コンテンツを一覧表示。<br>- 確認・削除・警告などのアクションを迅速に行えるUI。 |
