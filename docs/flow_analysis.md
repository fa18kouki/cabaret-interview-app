## 画像から抽出した主要要素

### 1. PoCユーザー（お客さん）フロー

#### 1.1 初期登録・認証
- **18歳以上（高校生不可）**
- **個人情報取得許諾**
- **ファクター認証（SMS認証等）**
- **空席検証窓口の記載**

#### 1.2 女の子とお店の連携
**SNSの連携:**
- LINE
- X (Twitter)
- Instagram
- Googleアカウント
- Appleアカウント

**メアドを取得して登録**

#### 1.3 個人情報管理者
- 個人情報管理者の選定
- 個人情報取得時の説明
- ファクター認証
- 空席検証窓口の記載

#### 1.4 メインフロー
1. **個人情報管理者リンク** → 個人情報取得許諾
2. **ファクター認証（SMS認証等）**
3. **空席検証窓口の記載**
4. **チャットボット・AIボイス**
   - 対話機能
   - 対話履歴・オフライン整形
5. **対話履歴・オフライン整形**
   - 対話履歴をテキスト化
   - ビッグクエリへ集積
   - ビッグクエリを基に統計分析
6. **公式の申請**
   - 身分証
   - プロフィール
   - 在留期間（該当者）

#### 1.5 お客さんとのマッチング
- **あなたにおすすめ！** → PoCのマッチング機能
- **料金一覧ここに！** → PoCのマッチング機能
- **料金マッチング情報提供など**
- **広告サービス**
- **会員施設**
- **人気女の子一覧**
- **アピールポイント**
  - 主婦で初上陸
  - 大学生、社会人、主婦
  - フォロワー数

#### 1.6 条件分岐
- **面接OK？**
  - YES → 面接の日時調整
  - NO → 次の候補を提案

#### 1.7 面接決定後
- **面接の日時調整＋オファー面接概要**
- **ニックネーム・連絡先・住所情報・正式なお店の詳細**
- **"無断欠席" ごとに次の候補者のリストイン**

#### 1.8 新規登録管理
- **新規登録管理者**
  - あまりにも"無断欠席"が多い場合、利用停止

#### 1.9 リスト管理
- **リストを手に入れることが目的**
  - 理由1: データ収集
  - 理由2: 営業活動

### 5. ビジネス要件

#### 5.1 目的
- **リスト獲得**: データ収集と営業活動

- **無断欠席削減**: ペナルティ機能による信頼性向上

#### 5.2 ユーザー種別
- **求職者（女性）**: 18歳以上、SNS連携、プロフィール登録
- **店舗（店長）**: 店舗情報登録、女の子募集、面接調整
- **個人情報管理者**: 個人情報の適切な管理

#### 5.3 収益モデル（推測）
