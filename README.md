# キャバクラ面接アプリ 要件定義書 v2

## 1. 概要

本ドキュメントは、キャバクラ面接アプリ（以下、本アプリ）の要件を定義するものです。本アプリは、キャバクラで働きたい女性（以下、キャスト）と店舗（以下、店舗）を効率的にマッチングさせ、面接プロセスを円滑にすることを目指します。

## 2. ユーザー種別と役割

本アプリには、以下の3つのユーザー種別が存在します。

| ユーザー種別 | 役割 |
| :--- | :--- |
| キャスト | 店舗を探し、応募・面接を行う女性ユーザー |
| 店舗 | キャストを募集し、採用活動を行う店舗ユーザー |
| 管理者 | システム全体を管理・監視するユーザー |

### 参考：キャストペルソナ

開発の参考として、以下に代表的なキャストのペルソナを記載します。

| ペルソナ | 特徴 |
| :--- | :--- |
| **👩 ケース①：六本木のエース級「レイナ様」** | 経験豊富で高いスキルを持つが、条件面での要求も高い。即戦力を求める店舗とのマッチングが期待される。 |
| **👧 ケース②：原石の女子大生「ヒナちゃん」** | 未経験だがポテンシャルを秘めている。育成環境の整った店舗とのマッチングが鍵となる。 |
| **👠 ケース③：条件ミスマッチの経験者「エリカ」** | 経験はあるものの、希望条件と市場のニーズに乖離がある。条件の見直しや新たな可能性の提案が求められる。 |

## 3. 機能要件

### 3.1. ユーザー登録・認証機能

| 機能 | 詳細 |
| :--- | :--- |
| **年齢確認** | 18歳以上（高校生不可）であることを確認するため、身分証明書のアップロードを必須とします。 |
| **個人情報取得許諾** | 登録時に、個人情報の取り扱いに関する規約への同意を必須とします。 |
| **アカウント作成** | SNSアカウント（LINE, X, Instagram, Google, Apple）またはメールアドレスでの登録を可能とします。 |
| **多要素認証** | セキュリティ向上のため、SMS認証などの多要素認証を導入します。 |

### 3.2. プロフィール機能

| 機能 | 詳細 |
| :--- | :--- |
| **キャストプロフィール** | 自己紹介文、写真、希望条件（勤務地、時給など）、マスト要件・ウォント要件を登録・編集できます。 |
| **店舗プロフィール** | 店舗情報（住所、営業時間、給与体系、福利厚生など）、求めるキャストのマスト要件・ウォント要件を登録・編集できます。 |

### 3.3. マッチング機能

| 機能 | 詳細 |
| :--- | :--- |
| **AIキャストランク判定** | 登録されたプロフィール情報や活動履歴に基づき、AIがキャストのランクを判定します。このランクは、マッチング精度の向上に利用されます。 |
| **キャスト検索** | 店舗は、希望条件（年齢、経験、ルックスなど）や、マスト・ウォント要件でキャストを検索できます。検索結果は、AIランク、人気順、新着順などでソート可能です。 |
| **店舗検索** | キャストは、希望条件（エリア、時給、待遇など）で店舗を検索できます。 |
| **おすすめ表示** | AIキャストランク判定ロジックを活用し、キャストと店舗の双方に、最適なマッチング候補をおすすめとして表示します。 |
| **スカウト機能** | 店舗は、気になるキャストにスカウトメッセージを送信できます。 |

### 3.4. コミュニケーション機能

| 機能 | 詳細 |
| :--- | :--- |
| **チャット機能** | マッチングが成立したキャストと店舗は、アプリ内でメッセージのやり取りができます。AIによるチャットボット機能も搭載し、初期対応を自動化します。 |
| **通話機能** | アプリ内で音声通話・ビデオ通話が可能です。 |

### 3.5. 面接管理機能

| 機能 | 詳細 |
| :--- | :--- |
| **面接予約** | アプリ上で面接の日程調整が可能です。 |
| **オファー管理** | 店舗は、採用したいキャストにアプリ経由で採用オファーを送信できます。 |
| **無断欠席対策** | 面接の無断欠席を繰り返すユーザーには、ペナルティ（一定期間の利用停止など）を課します。 |

### 3.6. 管理者機能

| 機能 | 詳細 |
| :--- | :--- |
| **ユーザー管理** | 全ユーザーの情報を閲覧・編集・削除できます。 |
| **コンテンツ管理** | 不適切なプロフィールや投稿を監視し、削除できます。 |
| **統計・分析** | 登録者数、マッチング数、面接数などの各種データを集計・分析し、レポートとして出力できます。AIマッチングの精度評価や改善のための分析機能も含まれます。 |

## 4. 非機能要件

| 項目 | 要件 |
| :--- | :--- |
| **パフォーマンス** | 全ての画面は3秒以内に表示されること。 |
| **セキュリティ** | 個人情報は暗号化して保存し、不正アクセス対策を徹底すること。 |
| **可用性** | サービスの稼働率は99.5%以上とすること。 |
| **UI/UX** | 直感的で分かりやすく、ストレスなく操作できるデザインであること。 |
